# Codex Execution Queue

## 목적
Claude 없이 Codex 단독으로 Ryan Members 자동화 기반을 연속 구현한다. 작업은 아래 순서대로 진행하며, 각 단계는 테스트 통과와 문서 갱신을 완료조건으로 한다.

## 공통 안전규칙
- 운영 Google Sheets 쓰기 금지
- Apps Script 배포 금지
- main 직접 수정 금지
- 학생명·전화번호·계좌정보를 로그와 fixture에 원문 저장 금지
- 자동확정·자동수정 금지
- 자격증명은 코드에 저장하지 않고 환경변수/GitHub Secrets만 사용
- 실패 시 다음 단계로 넘어가지 말고 원인과 재현 절차를 기록

## 실행 순서

### Phase 1 — 실데이터 Runner
1. Google Sheets read-only client 구현
2. 환경변수 기반 인증 로더 구현
3. 전체 Reader→Adapter→Reconciler→Report CLI 구현
4. JSON/TXT/CSV artifact 생성
5. dry-run 명령과 사용법 문서화

### Phase 2 — 검수 가능한 결과물
6. 오류 후보 최대 50건 추출
7. 정탐/오탐/판단보류 CSV 생성
8. 원천 시트명·행번호·규칙·근거·권장조치 포함
9. 개인정보 마스킹 검증 테스트 추가
10. GitHub Actions artifact 업로드

### Phase 3 — 탐지 정교화
11. 졸업 확정과 자동감지 알림 분리
12. 강사명이 학생명에 들어간 행 탐지
13. 동일 학생·날짜·금액·회차 중복 등록 탐지
14. 교재비·차액·취소 거래의 비패키지 분류
15. 가족입금·학생명-입금자명 불일치 예외 구조 설계
16. 동명이인 후보는 자동병합 금지

### Phase 4 — 운영 리포트
17. 소스별 행 수와 읽기 성공 여부
18. urgent/review/info 집계
19. 규칙별 발생 수와 상위 20건
20. 신규 발생과 기존 미해결 건 구분
21. 실행 이력 manifest 생성

### Phase 5 — 자동 실행
22. workflow_dispatch 실행
23. 매일 오전 7시 KST 스케줄 초안
24. secrets 미설정 시 안전 종료
25. artifact 보존기간 설정
26. 실패 시 issue/comment용 요약 생성

## Codex 작업 규칙
- 한 번에 하나의 이슈만 해결
- 각 이슈마다 별도 브랜치 사용 권장
- 구현 전 관련 파일과 테스트를 읽고 중복 구현 금지
- 변경 후 모든 Node 테스트 실행
- PR 본문에 변경점, 안전조건, 검증결과, 남은 한계를 기록
- 운영 시트 쓰기 또는 외부 메시지 발송이 필요한 단계는 구현하지 말고 인터페이스만 설계

## 최종 완료조건
- 읽기 전용 실데이터 실행이 한 명령으로 가능
- 산출물이 artifact로 남음
- 모든 테스트 통과
- 쓰기 권한과 쓰기 메서드가 존재하지 않음
- 운영자가 오류 후보 50건을 검수할 수 있음
